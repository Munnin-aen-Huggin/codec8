Hey HN,

I built Codec8 to solve a problem I've had for years: documentation debt.

We all know documentation is important, but it always ends up at the bottom of the to-do list. After yet another weekend spent writing READMEs, I decided to automate it.

**What it does:**
- Connect any GitHub repository (read-only OAuth)
- AI analyzes your codebase structure, dependencies, and key files
- Generates 4 types of docs: README, API documentation, architecture diagrams (Mermaid), and setup guides
- Edit inline, export markdown, or create a PR directly

**Tech stack:**
- SvelteKit + TypeScript
- Supabase (auth + database)
- Claude API for generation
- Stripe for payments
- Deployed on Vercel

**What I learned:**
- Context is everything. Generic prompts produce generic docs. I spent most of my time on the parsing logic to give Claude relevant context.
- Mermaid diagrams are surprisingly hard to generate correctly. Lots of iteration on the prompts.
- Markdown preview in a web editor is trickier than it looks (ended up using marked + DOMPurify).

**Pricing:**
Free for 1 repo, $99 lifetime deal for unlimited. No subscriptions - I hate those too.

Would love feedback, especially on:
1. What documentation formats would you add?
2. Any repos I can test it on? I'll share the output.

Try it: https://codec8.com

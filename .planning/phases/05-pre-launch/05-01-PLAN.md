# Phase 5.1: Beta Testing & Bug Fixes

## Objective
Get real user feedback on Codec8 by recruiting 10-20 beta testers, collecting structured feedback, and fixing critical bugs before launch.

## Prerequisites
- Production deployment working (Vercel)
- All core features complete (doc generation, payments, dashboard)
- Test accounts can be created

## Tasks

### 1. Create Beta Onboarding
**Files:** `src/routes/beta/+page.svelte`, `src/routes/beta/+page.server.ts`

- [ ] Create `/beta` landing page with signup form
- [ ] Collect: name, email, GitHub username, use case
- [ ] Store beta signups in Supabase `beta_signups` table
- [ ] Send confirmation email (or display confirmation message)

**Acceptance:** Beta signup page accessible, form submits to database

### 2. Beta User Recruitment
**Type:** Manual/Outreach

- [ ] Post in indie hacker communities (Twitter/X, Discord)
- [ ] Reach out to 5-10 developer contacts directly
- [ ] Offer free LTD tier for beta feedback
- [ ] Target: 10-20 active beta users

**Acceptance:** 10+ beta users signed up and actively testing

### 3. Feedback Collection System
**Files:** `src/routes/api/feedback/+server.ts`, `src/lib/components/FeedbackWidget.svelte`

- [ ] Create simple feedback widget (floating button)
- [ ] POST `/api/feedback` endpoint to store feedback
- [ ] Store: user_id, page, type (bug/feature/other), message, timestamp
- [ ] Add widget to dashboard pages

**Acceptance:** Users can submit feedback from any dashboard page

### 4. Bug Triage & Fixes
**Type:** Reactive based on feedback

- [ ] Create GitHub issues for reported bugs
- [ ] Prioritize: P0 (blocking), P1 (degraded), P2 (minor)
- [ ] Fix all P0 bugs before launch
- [ ] Fix P1 bugs if time permits

**Acceptance:** No P0 bugs remaining, P1 bugs documented

### 5. Critical Security Fixes (from CONCERNS.md)
**Files:** `src/lib/utils/license.ts`, `src/lib/server/stripe.ts`

- [ ] Replace `Math.random()` with `crypto.getRandomValues()` in license key generation
- [ ] Create `.env.example` with all required environment variables
- [ ] Add markdown sanitization to DocEditor (DOMPurify)

**Acceptance:** Security concerns addressed, .env.example exists

## Checkpoints

### Checkpoint 1: Beta Infrastructure Ready
- Beta signup page live
- Feedback widget deployed
- Database tables created

### Checkpoint 2: Beta Users Active
- 10+ beta users recruited
- Users actively generating docs
- Feedback coming in

### Checkpoint 3: Critical Fixes Complete
- Security fixes deployed
- P0 bugs fixed
- App stable for launch

## Estimated Scope
- Beta infrastructure: Small (2-3 hours)
- Recruitment: Medium (ongoing, 2-3 days)
- Bug fixes: Variable based on feedback
- Security fixes: Small (1-2 hours)

## Dependencies
- None (can start immediately)

## Risks
- Low beta signup rate: Mitigate with direct outreach
- Major bugs discovered: Prioritize ruthlessly, defer non-critical
